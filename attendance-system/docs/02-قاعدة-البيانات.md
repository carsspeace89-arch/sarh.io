## الجداول

### 1. جدول `branches` — الفروع

| العمود | النوع | القيد | الوصف |
|--------|-------|-------|-------|
| `id` | INT | PK, AUTO_INCREMENT | معرّف الفرع |
| `name` | VARCHAR(100) | UNIQUE, NOT NULL | اسم الفرع |
| `latitude` | DECIMAL(10,8) | NOT NULL | خط عرض موقع الفرع |
| `longitude` | DECIMAL(11,8) | NOT NULL | خط طول موقع الفرع |
| `geofence_radius` | INT | DEFAULT 50 | نصف قطر النطاق المسموح (متر) |
| `is_active` | TINYINT(1) | DEFAULT 1 | 1=مفعّل / 0=معطّل |
| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | تاريخ الإضافة |

### 2. جدول `employees` — الموظفون

| العمود | النوع | القيد | الوصف |
|--------|-------|-------|-------|
| `id` | INT | PK, AUTO_INCREMENT | معرّف فريد للموظف |
| `name` | VARCHAR(255) | NOT NULL | الاسم الكامل |
| `job_title` | VARCHAR(255) | NOT NULL | المسمى الوظيفي |
| `branch_id` | INT | FK → branches.id, NULL | الفرع التابع له |
| `pin` | VARCHAR(10) | UNIQUE, NOT NULL | الرقم السري للموظف |
| `phone` | VARCHAR(20) | NULL | رقم واتساب (اختياري) |
| `unique_token` | VARCHAR(64) | UNIQUE, NOT NULL | الرابط المشفر (hex 32 بايت = 64 حرف) |
| `device_fingerprint` | VARCHAR(64) | NULL | بصمة جهاز الموظف المسجّل (SHA-256) |
| `device_registered_at` | TIMESTAMP | NULL | تاريخ ووقت أول تسجيل للجهاز |
| `device_bind_mode` | TINYINT(1) | DEFAULT 0 | وضع الربط: 0=حر، 1=صارم (يمنع)، 2=مراقبة صامتة |
| `security_level` | INT | DEFAULT 2 | مستوى الأمان (مستقبلي) |
| `is_active` | TINYINT(1) | DEFAULT 1 | 1=مفعّل / 0=معطّل |
| `deleted_at` | TIMESTAMP | NULL | تاريخ الأرشفة (حذف ناعم — Soft Delete) |
| `created_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | تاريخ الإضافة |

**أوضاع ربط الجهاز (`device_bind_mode`):**
| القيمة | الوضع | السلوك |
|-------|------|--------|
| 0 | حر | الرابط يعمل من أي جهاز بدون ربط |
| 1 | صارم | يربط الجهاز ويمنع أي جهاز مختلف (شاشة قفل) |
| 2 | مراقبة صامتة | يسمح بالدخول لكن يسجل التلاعب بصمت |

**ملاحظة:** إذا كان للموظف `branch_id` محدد، يُستخدم نطاق الفرع الجغرافي. وإلا تُستخدم الإعدادات العامة.

### 6. جدول `known_devices` — الأجهزة المعروفة (تتبع ملكية الجهاز)

| العمود | النوع | القيد | الوصف |
|--------|-------|-------|-------|
| `id` | INT | PK, AUTO_INCREMENT | معرّف |
| `fingerprint` | VARCHAR(64) | NOT NULL | بصمة الجهاز (SHA-256) |
| `employee_id` | INT | FK → employees.id | الموظف الذي استخدم الجهاز |
| `usage_count` | INT | DEFAULT 1 | عدد مرات الاستخدام |
| `first_used_at` | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | أول استخدام |
| `last_used_at` | TIMESTAMP | ON UPDATE CURRENT_TIMESTAMP | آخر استخدام |

**UNIQUE KEY:** `(fingerprint, employee_id)` — كل جهاز+موظف سطر واحد.  
**ملكية الجهاز:** الموظف صاحب أعلى `usage_count` لنفس الـ `fingerprint` يُعتبر مالك الجهاز.

## استعلامات شائعة مرجعية

```sql
-- إحصائيات اليوم
SELECT
  (SELECT COUNT(DISTINCT employee_id) FROM attendances WHERE DATE(timestamp) = CURDATE() AND type = 'in') AS checked_in,
  (SELECT COUNT(DISTINCT employee_id) FROM attendances WHERE DATE(timestamp) = CURDATE() AND type = 'out') AS checked_out,
  (SELECT COUNT(*) FROM employees WHERE is_active = 1) AS total;

-- موظفي فرع معين
SELECT e.*, b.name AS branch_name FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE b.id = ?;

-- حضور موظف معين اليوم
SELECT type, timestamp FROM attendances
WHERE employee_id = ? AND DATE(timestamp) = CURDATE()
ORDER BY timestamp ASC;

-- الغائبون اليوم
SELECT e.name, e.job_title, b.name AS branch_name FROM employees e
LEFT JOIN branches b ON e.branch_id = b.id
WHERE e.is_active = 1
AND e.id NOT IN (
  SELECT DISTINCT employee_id FROM attendances WHERE DATE(timestamp) = CURDATE()
);

-- تقرير موظف لفترة محددة
SELECT a.type, a.timestamp, a.latitude, a.longitude
FROM attendances a
WHERE a.employee_id = ? AND a.timestamp BETWEEN ? AND ?
ORDER BY a.timestamp;
```
