## 1. نظرة عامة على التطبيق

### الغرض
نظام حضور وانصراف إلكتروني متكامل يعتمد على **الموقع الجغرافي (Geofencing)** و**بصمة الجهاز الرقمية (Device Fingerprinting)**. يُمكّن المشرفين من متابعة حضور الموظفين لحظياً، ويسمح للموظفين بتسجيل دخولهم وانصرافهم عبر روابط فريدة دون الحاجة لتطبيق مخصص.

### الفئة المستهدفة
- **المشرفون/المديرون:** إدارة الموظفين، مراقبة الحضور، استعراض التقارير، كشف التلاعب.
- **الموظفون:** تسجيل الحضور والانصراف عبر رابط فريد يُفتح من متصفح الهاتف.

### التقنيات المستخدمة

| الطبقة | التقنية |
|--------|---------|
| الخادم | PHP 8.1+ (Procedural + OOP) |
| قاعدة البيانات | MySQL 8.0+ (InnoDB, utf8mb4_unicode_ci) |
| الواجهة الأمامية | HTML5, CSS3 (متغيرات CSS), Vanilla JavaScript (ES6+) |
| الخرائط | Leaflet.js (خريطة تفاعلية) |
| الرسم | Canvas API (رادار GPS مخصص) |
| الصوت | Web Audio API (تسجيل + مؤثرات صوتية) |
| الموقع الجغرافي | Geolocation API + صيغة Haversine |
| الاتصال بقاعدة البيانات | PDO مع Prepared Statements حقيقية |
| خادم الويب | Apache 2.4+ (mod_rewrite) |
| النشر | SSH + SCP (PuTTY) → Hostinger |
| إدارة الإصدارات | Git + GitHub |

### ملاحظات عامة
- **بدون إطار عمل (Framework):** التطبيق مبني بالكامل بـ PHP خام بدون Laravel أو أي إطار، مما يمنح خفة كبيرة لكن يزيد المسؤولية الأمنية.
- **بدون Composer:** لا اعتمادات خارجية — كل الشيفرة مكتوبة يدوياً.
- **PWA جزئي:** يدعم manifest.json وService Worker للعمل كتطبيق ويب متقدم.

## 3. تحليل قاعدة البيانات

### الجداول (10 جداول)

| # | الجدول | الغرض | السجلات المتوقعة |
|---|--------|-------|-----------------|
| 1 | `branches` | الفروع الجغرافية | عشرات |
| 2 | `employees` | بيانات الموظفين | مئات |
| 3 | `attendances` | سجلات الحضور والانصراف | آلاف/يومياً |
| 4 | `admins` | المشرفون | أقل من 10 |
| 5 | `settings` | إعدادات النظام (Key-Value) | عشرات |
| 6 | `known_devices` | تتبع الأجهزة وملكيتها | مئات |
| 7 | `tampering_cases` | حالات التلاعب المكتشفة | متفاوتة |
| 8 | `leaves` | الإجازات | متفاوتة |
| 9 | `audit_log` | سجل العمليات الإدارية | آلاف |
| 10 | `login_attempts` | محاولات الدخول | مؤقتة (تُنظّف دورياً) |

### تحليل التصميم

#### نقاط القوة
- **الفهارس (Indexes):** فهارس محددة على الأعمدة الأكثر استعلاماً:
  - `attendances`: فهارس على `(employee_id, timestamp)`, `(type, timestamp)`, `(attendance_date)`
  - `known_devices`: فهارس على `fingerprint`, `employee_id`, ومفتاح فريد مركب `(fingerprint, employee_id)`
  - `login_attempts`: فهرس على `(ip_address, attempted_at)`
- **العلاقات الأجنبية (Foreign Keys):**
  - `employees.branch_id → branches.id (ON DELETE SET NULL)` ← حماية عند حذف الفرع
  - `attendances.employee_id → employees.id (ON DELETE CASCADE)` ← حذف السجلات مع الموظف
  - `known_devices.employee_id → employees.id (ON DELETE CASCADE)`
  - `leaves` يرتبط بـ `employees` و `admins`
- **الحذف الناعم (Soft Delete):** عمود `deleted_at` في `employees` — يحافظ على البيانات التاريخية.
- **ENUM للبيانات المحدودة:** `attendances.type` محصور في `('in', 'out', 'overtime')`, `leaves.status` في `('pending', 'approved', 'rejected')`.
- **أعمدة التدقيق:** `created_at` موجود في جميع الجداول, `ip_address` و `user_agent` في `attendances`.

#### مستوى التطبيع (Normalization)
- **3NF متحقق:** لا توجد تكرارات واضحة. الإعدادات في جدول Settings منفصل (Key-Value).
- **ملاحظة:** جدول `branches` يحوي أعمدة مواعيد (12 عموداً زمنياً) — يمكن فصلها إلى جدول `branch_schedules` إذا كانت هناك ورديات متعددة لكل فرع.

#### نقاط الضعف
| المشكلة | الخطورة | التوصية |
|---------|---------|---------|
| لا يوجد جدول `tampering_cases` ضمن `install.php` بشكل صريح | ⚠️ متوسطة | `verify-device.php` ينشئه ديناميكياً بـ `CREATE TABLE IF NOT EXISTS` — يُفضّل إنشاؤه في المثبّت |
| `secret_reports` غير موجود في `install.php` | ⚠️ متوسطة | يجب إنشاؤه في سكربت التثبيت |
| لا يوجد حد أقصى لحجم `audit_log` | 🟡 منخفضة | إضافة مهمة تنظيف دورية أو تقسيم (partitioning) |
| `employees.pin` بحد أقصى 10 أحرف — TEXT | 🟡 منخفضة | كافٍ كرقم تعريفي داخلي |
| `attendances.user_agent` من نوع `TEXT` | 🟡 منخفضة | يمكن تحويله إلى `VARCHAR(512)` لتوفير المساحة |

## 4. تحليل الأمان

### 4.1 نقاط القوة الأمنية

#### مصادقة المشرف
| الآلية | التقييم | التفاصيل |
|--------|---------|---------|
| تشفير كلمة المرور | ✅ ممتاز | `password_hash()` + `password_verify()` (bcrypt) |
| CSRF Protection | ✅ ممتاز | Token لمرة واحدة (One-Time Token) مع `hash_equals()` + تدوير بعد كل استخدام |
| حماية الجلسة | ✅ ممتاز | `session_regenerate_id(true)` عند الدخول |
| إعدادات الكوكي | ✅ ممتاز | `httponly=1`, `samesite=Strict`, `cookie_secure` ديناميكي حسب HTTPS |
| حماية Brute Force | ✅ ممتاز | 5 محاولات / 10 دقائق — يعتمد على DB + fallback للـ Session |
| سجل التدقيق | ✅ ممتاز | `auditLog()` يسجل كل عملية إدارية مع IP |

#### مصادقة الموظف
| الآلية | التقييم | التفاصيل |
|--------|---------|---------|
| Token الفريد | ✅ ممتاز | 64 حرف hex = `bin2hex(random_bytes(32))` — ٢²⁵⁶ احتمال |
| بصمة الجهاز | ✅ ممتاز | SHA-256 + `hash_equals()` + 3 أوضاع ربط |
| كشف التلاعب | ✅ ممتاز | المراقبة الصامتة (وضع 2) ابتكار ممتاز |

#### حماية API
| الآلية | التقييم | التفاصيل |
|--------|---------|---------|
| Rate Limiting | ✅ جيد | 30 طلب/دقيقة مع قفل مضاعف — يعتمد على الملفات (متوافق مع الاستضافة المشتركة) |
| CORS | ✅ جيد | `Access-Control-Allow-Origin: [SITE_URL]` محدد |
| طريقة الطلب | ✅ جيد | كل API يتحقق من `POST` فقط |
| التحقق من المدخلات | ✅ جيد | `sanitize()` + `trim()` + Type Casting |
| SQL Injection | ✅ ممتاز | Prepared Statements في كل مكان + `EMULATE_PREPARES = false` |
| XSS | ✅ جيد | `htmlspecialchars()` على المخرجات + `sanitize()` على المدخلات |

#### حماية الملفات
| الآلية | التقييم | التفاصيل |
|--------|---------|---------|
| install.php | ✅ جيد | ملف قفل `install.lock` + حماية `.htaccess` |
| includes/ | ✅ جيد | حماية بـ `.htaccess` |
| رسائل الخطأ | ✅ جيد | `display_errors = 0` + `log_errors = 1` |

### 4.2 نقاط الضعف الأمنية

| # | الثغرة | الخطورة | التفاصيل | التوصية |
|---|--------|---------|---------|---------|
| 1 | **Token في URL (GET parameter)** | 🔴 عالية | Token الموظف يُمرّر في `?token=...` — يُخزّن في سجلات الخادم، تاريخ المتصفح، Referer headers | ✅ استخدام POST أو تخزين Token في Cookie/LocalStorage بعد الدخول الأول |
| 2 | **Cron secret في URL** | 🟠 متوسطة | `auto-checkout.php?secret=auto_checkout_2026` — كلمة ثابتة قابلة للتخمين | ✅ استخدام متغير بيئة أو تقييد بـ IP أو CLI فقط |
| 3 | **Rate Limiter بالملفات** | 🟡 منخفضة | يعتمد على `sys_get_temp_dir()` — قد لا يعمل في بعض بيئات الاستضافة المشتركة مع حماية open_basedir | ✅ استخدام جدول DB كبديل |
| 4 | **تسرب معلومات install.php** | 🟡 منخفضة | بيانات الاتصال مكررة (hardcoded defaults + .env) | ✅ حذف الملف بعد التثبيت أو الاكتفاء بـ .env |
| 5 | **غياب Content-Security-Policy** | 🟡 منخفضة | لا يوجد CSP header في الاستجابات | ✅ إضافة `Content-Security-Policy` لمنع حقن السكربتات |
| 6 | **التقارير السرية بدون مصادقة إضافية** | 🟡 منخفضة | أي شخص لديه Token الموظف يستطيع إرسال بلاغ | ⚠️ مقبول — التصميم يعتمد على Token كهوية |
| 7 | **SSH credentials في الكود/الأوامر** | 🟠 متوسطة | كلمة مرور SSH مكشوفة في أوامر النشر | ✅ استخدام SSH keys بدلاً من كلمات المرور |

```
POST → Method Check → JSON Parse → Token Validation
  → Time Window Check → Geofence Check → Duplicate Check → Record
```
> تسلسل منطقي وآمن — يفشل مبكراً (Fail-Fast).

### ملاحظة على verify-device
التنفيذ يحتوي على ذكاء ملحوظ:
- **كشف التسجيل بالنيابة (Proxy Check-in):** يتحقق هل نفس البصمة مربوطة بموظف آخر.
- **تتبع ملكية الجهاز:** عبر `known_devices` و `usage_count`.
- **3 أوضاع مرنة:** حر / صارم / مراقبة صامتة — تصميم ذكي يناسب سيناريوهات مختلفة.

## 7. الميزات الرئيسية

### 7.1 نظام الجيوفينس
- **صيغة Haversine:** حساب دقيق للمسافة على سطح الكرة الأرضية — يتم على الخادم (لا يمكن تزويره من المتصفح).
- **دعم الفروع المتعددة:** كل فرع بإحداثيات ونصف قطر مستقلين.
- **تدريج ذكي:** إذا لم يُحدد فرع، يُستخدم الإعدادات العامة. إذا لم يُحدد موقع أصلاً، يُسمح.

### 7.2 بصمة الجهاز (Device Fingerprinting)
- **5 مكونات:** UserAgent + أبعاد الشاشة + المنطقة الزمنية + اللغة + Canvas fingerprint.
- **3 أوضاع ربط:** حر (0) / صارم (1) / مراقبة صامتة (2) — تصميم مرن وذكي.
- **تتبع الملكية:** جدول `known_devices` يتتبع عدد مرات الاستخدام ويحدد المالك.
- **كشف التسجيل بالنيابة:** إذا استُخدم نفس الجهاز لموظفين مختلفين، يُسجل `proxy_checkin`.
- **حماية من Timing Attacks:** استخدام `hash_equals()` بدلاً من `===`.

### 7.3 الروابط الفريدة
- 64 حرف hex (256-bit entropy) — مستحيل التخمين.
- قابلة للتجديد فردياً أو جماعياً.
- مُرسلة عبر واتساب بنقرة واحدة.

### 7.4 الدوام الإضافي
- تفعيل/تعطيل عام أو لكل فرع.
- ضوابط: يبدأ بعد X دقيقة من الانصراف، حد أدنى للمدة.
- يتطلب تسجيل انصراف أولاً.

### 7.5 نظام الورديات المزدوجة
- وردية صباحية + مسائية مع كشف تلقائي.
- دعم تجاوز منتصف الليل (midnight crossing).
- الأنصراف التلقائي عبر Cron (`auto-checkout.php`).

### 7.6 البلاغات السرية
- **نص + صور + تسجيل صوتي** مع مؤثرات تغيير الصوت.
- 4 مؤثرات: عادي / عميق / حاد / روبوت — لحماية هوية المُبلِّغ.
- أنواع البلاغات: عام / مخالفة / تحرش / سرقة / سلامة / أخرى.

### 7.7 الانصراف التلقائي (Cron)
- يفحص كل دقيقة.
- يدعم الورديتين ويكشف الوردية عبر وقت الدخول.
- يدعم تجاوز منتصف الليل.
- حماية CLI + secret key.

### 7.8 التحديث اللحظي (Real-time)
- **لوحة التحكم:** AJAX polling يحدّث الإحصائيات وآخر التسجيلات لحظياً.
- **صفحة الحضور:** بيانات الحضور تتحدث بدون refresh.
- **مؤشر حي (Live Indicator):** نقطة انميشن خضراء تشير لاتصال نشط.

## 8. نقاط القوة

### 8.1 التصميم والمعمارية
1. **خفة التشغيل:** بدون أطر عمل ثقيلة — كل طلب PHP ينتهي في أجزاء من الثانية.
2. **هيكل ملفات واضح:** فصل نظيف بين API وAdmin وEmployee وIncludes.
3. **توثيق استثنائي:** 8 ملفات توثيق عربية شاملة — نادر في المشاريع بهذا الحجم.
4. **Singleton Pattern:** اتصال PDO واحد لكل طلب — يمنع تسرب الاتصالات.
5. **Settings Cache:** `loadAllSettings()` يحمّل كل الإعدادات دفعة واحدة مع static caching.

### 8.2 الأمان
6. **طبقات أمنية متعددة:** bcrypt + CSRF (One-Time) + Rate Limiting + Brute Force Protection + Device Binding + Geofencing.
7. **المراقبة الصامتة (وضع 2):** ابتكار ذكي يسمح بكشف التلاعب دون إنذار الموظف.
8. **سجل تدقيق شامل:** كل عملية إدارية تُسجّل مع IP ورقم المشرف.
9. **IP Spoofing Protection:** `getClientIP()` يثق فقط بـ proxies محددة.
10. **حماية تسرب معلومات DB:** رسالة خطأ عامة عند فشل الاتصال (`display_errors=0`).

### 8.3 تجربة المستخدم
11. **صفر تثبيت:** الموظف يفتح رابطاً فقط — لا تطبيق، لا تسجيل دخول.
12. **تصميم Mobile-First:** واجهة الموظف مصممة للهاتف حصراً.
13. **الرادار التفاعلي:** تجربة بصرية فريدة مع Canvas Animation.
14. **دعم عربي كامل:** RTL + خط Tajawal + أيقونات SVG مدمجة.
15. **PWA:** قابل للتثبيت كتطبيق على الشاشة الرئيسية.

### 8.4 الذكاء البرمجي
16. **كشف الوردية التلقائي:** يحدد الوردية بناءً على الوقت الحالي أو وقت الدخول.
17. **دعم تجاوز منتصف الليل:** منطق متين في check-in/check-out/auto-checkout.
18. **الإنشاء التلقائي للجداول:** `CREATE TABLE IF NOT EXISTS` في verify-device — يعمل حتى بدون تشغيل المثبت.
19. **Auto-detect SITE_URL:** يكتشف البروتوكول والمسار تلقائياً — لا حاجة لتعديل يدوي.
20. **تغيير الصوت بـ Web Audio API:** تنفيذ كامل للتحويل الصوتي على المتصفح بدون خادم.
