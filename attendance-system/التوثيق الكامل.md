# نظام الحضور والانصراف الإلكتروني

**الإصدار:** 3.2.0 | **آخر تحديث:** مارس 2026

نظام ويب متكامل لتسجيل حضور وانصراف الموظفين عبر روابط فريدة (WhatsApp) مع التحقق من الموقع الجغرافي، بصمة الجهاز، والتحديث بالوقت الفعلي.

## المميزات الرئيسية

| الميزة | الوصف |
|--------|-------|
| تسجيل الحضور/الانصراف | عبر روابط فريدة لكل موظف (WhatsApp) |
| التحقق الجغرافي (Geofencing) | التأكد من تواجد الموظف ضمن نطاق العمل |
| بصمة الجهاز (Device Fingerprinting) | التحقق من الجهاز المسجل لكل موظف |
| إدارة الفروع | فروع متعددة بمواعيد وإحداثيات مستقلة |
| الدوام الإضافي | تتبع وإدارة ساعات العمل الإضافية |
| التحديث بالوقت الفعلي | تحديث تلقائي للبيانات كل 15-20 ثانية عبر AJAX |
| مؤشر البث المباشر | مؤشر مرئي يوضح حالة الاتصال (مباشر / متوقف / خطأ) |
| تصدير CSV | تصدير تقارير الحضور بصيغة CSV |
| خريطة تفاعلية | عرض مواقع الحضور على خريطة Leaflet |
| واجهة عربية RTL | تصميم احترافي بالكامل باللغة العربية |
| SVG أيقونات | أيقونات متجهة مدمجة بدلاً من Emoji |
| متجاوب (Responsive) | يعمل على الهاتف والجهاز اللوحي والحاسب |
| 🆕 شعار مخصص | الشعار (loogo.png) في كل مكان مع أنيميشن (نبض، توهج، تعويم) |
| 🆕 خلفية مزخرفة | نمط الشعار المتكرر كعلامة مائية متحركة |
| 🆕 صوت الرادار | صوت سونار عند فتح صفحة الموظف (Web Audio API) |
| 🆕 خريطة الفرع | خريطة Leaflet تُظهر مركز العمل مع دائرة النطاق (+10م) |
| 🆕 تنبيه GPS مستمر | نافذة منبثقة مستمرة إذا لم يُفعّل الموقع |
| 🆕 لوحة تحكم متجاوبة | تصميم محسّن للهاتف مع زر خروج مرئي |
| 🆕 PWA | تطبيق ويب تقدمي مع Service Worker |
| 🆕 نظام الأمان المتقدم | Rate Limiting، CSRF، حماية Brute Force |
| 🆕 سجل التدقيق | تسجيل كل العمليات في audit_log |
| 🆕 الحذف الناعم | Soft delete للموظفين (deleted_at) |
| 🆕 انصراف تلقائي | تسجيل انصراف تلقائي عند نهاية وقت الانصراف |
| 🆕 نظام دوام إضافي محسّن | session-based مع مؤقت مباشر وإنهاء تلقائي عند 3 فجراً |
| 🆕 تقرير التأخير | حساب دقيق للتأخير على مستوى الدقيقة مع فلاتر متقدمة |
| 🆕 ربط الجهاز يدوي | الربط يتم فقط عندما يُفعّله المشرف يدوياً (بدون ربط تلقائي) |
| 🆕 تجديد جماعي للروابط | زر واحد لتجديد جميع روابط الموظفين دفعة واحدة |
| 🆕 إرسال جماعي واتساب | إرسال جميع الروابط برسالة واحدة عبر واتساب |
| 🆕 GPS Banner | شريط تنبيه شفاف غير معطل للصفحة |

## التقنيات المستخدمة

- **PHP 8.2+** Native (بدون إطارات عمل)
- **MySQL 8.0** / MariaDB
- **JavaScript** خالص (ES6+ بدون إطارات)
- **HTML5 + CSS3** مع تصميم RTL عربي
- **Geolocation API** للتحقق الجغرافي
- **Web Audio API** لصوت الرادار/السونار
- **Fetch API** للتحديث بالوقت الفعلي (AJAX Polling)
- **Leaflet.js** للخرائط التفاعلية (واجهة الموظف + الإدارة)
- **Service Worker + PWA** للتطبيق التقدمي
- **Apache / LiteSpeed** مع mod_rewrite
- **SVG** للأيقونات (بدون مكتبات خارجية)

يحتاج النظام إلى cron job يعمل كل دقيقة لتنفيذ المهام الآلية التالية:

### 1. الانصراف التلقائي
يسجل انصراف تلقائي للموظفين الذين لم يسجلوا انصراف عند وصول وقت `check_out_end_time` المحدد في جدول الفرع.

### 2. إنهاء الدوام الإضافي التلقائي
ينهي sessions الدوام الإضافي المفتوحة تلقائياً عند الساعة 3:00 فجراً (موعد حفظ السجلات).

#### cPanel / Hostinger
1. افتح لوحة تحكم cPanel → Cron Jobs
2. أضف cron job جديد:
   - الدقائق: `*` (كل دقيقة)
   - الساعات: `*`
   - أيام الشهر: `*`
   - الأشهر: `*`
   - أيام الأسبوع: `*`
   - الأمر:
     ```bash
     /usr/bin/php /path/to/attendance-system/cron/auto-checkout.php
     ```

#### LiteSpeed / Apache (crontab)
```bash
crontab -e
```
أضف السطر التالي:
```
* * * * * /usr/bin/php /home/username/public_html/attendance-system/cron/auto-checkout.php >> /home/username/cron.log 2>&1
```

#### للتشغيل اليدوي (اختبار)
```bash

```

أو عبر HTTP (لأغراض الاختبار فقط):

## التحديث بالوقت الفعلي (Real-time Updates)

### آلية العمل
يستخدم النظام تقنية **AJAX Long Polling** عبر `fetch()` API لتحديث البيانات تلقائياً:

```
[المتصفح] ──fetch()──> [api/realtime-dashboard.php] ──SQL──> [MySQL]
     ↑                          │
     └───── JSON Response ──────┘
         (كل 15-20 ثانية)
```

### الصفحات المدعومة

| الصفحة | API Endpoint | فترة التحديث | البيانات المحدثة |
|--------|-------------|-------------|-----------------|
| لوحة التحكم | `/api/realtime-dashboard.php` | 15 ثانية | الإحصائيات + آخر التسجيلات + الغائبون |
| تقارير الحضور | `/api/realtime-attendance.php` | 20 ثانية | الإحصائيات + السجلات + الترقيم |

### مؤشر البث المباشر
يظهر مؤشر مرئي في الزاوية يوضح حالة الاتصال:
- **نقطة خضراء نابضة** + "مباشر": الاتصال نشط والبيانات محدثة
- **نقطة صفراء ثابتة** + "متوقف": الصفحة مخفية أو خطأ مؤقت
- **نقطة حمراء ثابتة** + "خطأ": فشل الاتصال 3 مرات متتالية

### تحسينات الأداء
- **توقف تلقائي**: يتوقف الاستطلاع عند إخفاء التبويب (`visibilitychange`)
- **استئناف فوري**: يُحدّث فوراً عند العودة للصفحة
- **عدد الأخطاء**: يتتبع عدد الأخطاء المتتالية قبل إظهار حالة "خطأ"
- **No-cache**: ترويسات عدم التخزين المؤقت لضمان بيانات طازجة
- **تحريك سلس**: تأثيرات CSS عند تغيّر القيم (flash animation)

### التخصيص
لتغيير فترة التحديث، عدّل المتغير في الصفحة:
```javascript
const REFRESH_INTERVAL = 15000; // بالميللي ثانية (15 ثانية)
```

1. سجّل دخولك كمدير

- مواعيد دوام مستقلة (بداية/نهاية)

```
إحداثيات الفرع (إذا كان الموظف معين لفرع)

    ↓ fallback
مسموح بأي موقع (إذا كلا الإحداثيات = 0)
```

## بصمة الجهاز (Device Fingerprinting)

### آلية العمل (ربط يدوي فقط)
1. عند فتح الموظف لرابطه، يتم حساب بصمة الجهاز وإرسالها للخادم
2. **إذا لم يكن هناك ربط مفعّل:** الموظف يدخل بدون ربط جهاز
3. **إذا فعّل المشرف وضع الربط:** يُربط الجهاز عند الدخول التالي
4. بعد الربط، كل تسجيل لاحق يتحقق من بصمة الجهاز
5. إذا اختلفت البصمة، يُرفض الدخول

> **لا يحدث ربط تلقائي** — المشرف يتحكم بالكامل.

### إدارة الأجهزة
- **فك الربط:** يمكن للمشرف فك ربط الجهاز (يصبح الرابط حراً)
- **ربط جهاز جديد:** فك الربط أولاً ثم تفعيل وضع الربط
- **الربط اليدوي:** من صفحة الموظفين → زر 🔑 (ينتظر الدخول التالي)

## API Reference

### POST `/api/check-in.php` — تسجيل الحضور
**الطلب:**
```json
{
  "token": "abc123def456...",
  "latitude": 24.7136,
  "longitude": 46.6753,
  "accuracy": 10,
  "fingerprint": "device_fingerprint_hash"
}
```
**الرد (نجاح):**
```json
{
  "success": true,
  "message": "تم تسجيل الدخول بنجاح",
  "distance": 45,
  "time": "08:15 AM"
}
```
**الرد (خطأ):**
```json
{
  "success": false,
  "message": "أنت خارج نطاق العمل! المسافة: 350 متر (الحد المسموح: 200 متر)"
}
```
**شروط التسجيل:**
- الموظف نشط (`is_active = 1`)
- ضمن وقت تسجيل الحضور (`check_in_start_time` → `check_in_end_time`)
- ضمن نطاق السياج الجغرافي
- بصمة الجهاز مطابقة (أو أول تسجيل)
- لم يُسجل خلال آخر 5 دقائق

### POST `/api/check-out.php` — تسجيل الانصراف
**الطلب:** نفس بنية check-in  
**شروط إضافية:**
- وجود تسجيل حضور اليوم
- ضمن وقت تسجيل الانصراف (`check_out_start_time` → `check_out_end_time`)

### POST `/api/ot.php` — الدوام الإضافي
**الطلب:** نفس بنية check-in  
**شروط إضافية:**
- `allow_overtime = 1` في إعدادات الفرع أو النظام
- وجود تسجيل انصراف اليوم

### لوحة التحكم (`dashboard.php`)
- **5 بطاقات إحصائية**: الفروع النشطة، إجمالي الموظفين، الحاضرون، المنصرفون، الغائبون
- **آخر 15 تسجيل**: جدول يعرض أحدث الدخول/الانصراف
- **الغائبون اليوم**: قائمة أول 10 موظفين غائبين
- **تحديث تلقائي** كل 15 ثانية مع مؤشر "مباشر"
- **تأثيرات بصرية**: وميض عند تغيّر القيم، انزلاق للسجلات الجديدة

### إدارة الفروع (`branches.php`)
- **إضافة / تعديل / حذف** فروع
- **تفعيل / تعطيل** فروع
- كل فرع يتضمن:
  - اسم + إحداثيات + نصف قطر جيوفينس
  - مواعيد الدوام (بداية/نهاية)
  - أوقات تسجيل الحضور والانصراف
  - إعدادات الدوام الإضافي
  - عدد الموظفين المعينين

### إدارة الموظفين (`employees.php`)
- **إضافة / تعديل / حذف** موظفين
- **تفعيل / تعطيل** حسابات
- **إعادة توليد** رابط الحضور (token جديد)
- **إعادة تعيين** بصمة الجهاز
- **نسخ** رابط الحضور
- **إرسال** عبر واتساب
- تعيين موظف لفرع

### تقارير الحضور (`attendance.php`)
- **فلاتر متقدمة**: تاريخ من/إلى، موظف، فرع، نوع (دخول/انصراف)
- **4 بطاقات إحصائية** للفترة المحددة
- **تصدير CSV** بصيغة Excel-compatible (مع BOM UTF-8)
- **أزرار خريطة** لعرض موقع كل تسجيل
- **ترقيم الصفحات** (25 سجل/صفحة)
- **تحديث تلقائي** كل 20 ثانية مع مؤشر "مباشر"

### تقرير التأخير (`late-report.php`) 🆕
- **فلاتر متقدمة**: موظف محدد / فرع / جميع الموظفين + فترة زمنية
- **حساب دقيق للتأخير**: على مستوى الدقيقة بمقارنة وقت check-in مع check_in_end_time
- **4 بطاقات إحصائية**: مجموع التأخير، عدد أيام التأخير، الموظفون المتأخرون، متوسط التأخير

- **تقرير تفصيلي يومي**: التاريخ، الموظف، الفرع، وقت الحضور، الوقت المتوقع، مدة التأخير
- **تصنيف ملون**: تأخير منخفض (<15د) / متوسط (15-45د) / عالي (>45د)

## استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| الموقع لا يُحدَّد | تأكد من HTTPS (أو localhost)، وأن المتصفح يسمح بالوصول للموقع |
| خطأ في قاعدة البيانات | تحقق من `includes/config.php` وصلاحيات المستخدم |
| صفحات 404 | تأكد من تفعيل mod_rewrite وأن `.htaccess` مفعّل |
| بصمة الجهاز مرفوضة | أعد تعيين بصمة الجهاز من إدارة الموظفين |
| مؤشر "مباشر" يظهر "خطأ" | تحقق من الاتصال بالخادم واتصال قاعدة البيانات |
| لا يعمل على الموبايل | تأكد من استخدام HTTPS (مطلوب لـ Geolocation API) |
| الانصراف لا يظهر | تحقق من إعدادات `checkout_show_before` و `check_out_start_time` |
| الدوام الإضافي لا يعمل | تأكد من `allow_overtime = 1` وأن شروط الوقت محققة |
| التحديث التلقائي بطيء | عدّل `REFRESH_INTERVAL` في JavaScript (الافتراضي 15-20 ثانية) |

## حساب بصمة الجهاز (JavaScript)

```javascript
async function getFingerprint() {
  const parts = [
    navigator.userAgent,
    screen.width + 'x' + screen.height + 'x' + screen.colorDepth,
    Intl.DateTimeFormat().resolvedOptions().timeZone,
    navigator.language,
    navigator.platform || ''
  ];
  // Canvas fingerprint — بصمة رسومية فريدة
  const c = document.createElement('canvas');
  const x = c.getContext('2d');
  x.font = '14px Arial'; x.fillText('fp', 2, 2);
  parts.push(c.toDataURL().slice(-50));

  const raw = parts.join('|');
  const buf = await crypto.subtle.digest('SHA-256', new TextEncoder().encode(raw));
  return Array.from(new Uint8Array(buf))
    .map(b => b.toString(16).padStart(2, '0')).join('');
}
```

# الأمان — التوثيق الكامل

## طبقات الحماية المنفّذة

---

### 1. نظام المصادقة

#### مشرف النظام
- كلمة مرور مشفّرة بـ **bcrypt** (`password_hash` + `password_verify`)
- **CSRF Token** على جميع نماذج POST — يمنع هجمات Cross-Site Request Forgery
- تجديد معرّف الجلسة عند كل تسجيل دخول (`session_regenerate_id(true)`)
- الجلسة تنتهي بإغلاق المتصفح (`session.cookie_lifetime = 0`)
- `session.cookie_httponly = 1` ← JavaScript لا يستطيع قراءة الكوكي
- `session.cookie_samesite = Strict` ← حماية إضافية من CSRF

#### الموظف
- لا يوجد كلمة مرور — الهوية عبر **unique_token** (64 حرف عشوائي)
- التوكن يُنتج بـ `bin2hex(random_bytes(32))` ← آمن كريبتوغرافياً
- لا يمكن تخمينه: احتمال 1 من 2^256

---

### 2. قفل الجهاز (Device Fingerprinting)

#### آلية العمل (الربط يدوي فقط — v3.2.0)
```
الدخول الأول (بدون ربط مفعّل):
  JS يحسب بصمة SHA-256 ← ترسل للـ API
  ← لا توجد بصمة محفوظة + وضع الربط غير مفعّل
  ← الموظف يدخل بدون ربط ✅

تفعيل الربط من المشرف:
  المشرف يضغط "ربط الجهاز" ← device_bind_mode = 1
  ← الموظف يفتح الرابط ← الجهاز يُربط تلقائياً ✅

الدخول التالي على نفس الجهاز (مربوط):
  JS يحسب نفس البصمة ← hash_equals() ← ✅

دخول من جهاز مختلف (مربوط):
  JS يحسب بصمة مختلفة ← hash_equals() يفشل ← ❌ رسالة قفل
```

> **ملاحظة:** لا يحدث ربط تلقائي. المشرف يتحكم بالكامل متى يتم ربط جهاز الموظف.

#### مكونات البصمة
1. `userAgent` — نظام التشغيل والمتصفح
2. `screen.width × height × colorDepth` — مواصفات الشاشة
3. `Intl timezone` — المنطقة الزمنية
4. `navigator.language` — لغة الجهاز
5. `canvas fingerprint` — بصمة رسومية فريدة للـ GPU

#### مقاومة التزوير
- `hash_equals()` في PHP ← يمنع **Timing Attacks**
- البصمة SHA-256 لا تُعكس ← المهاجم لا يستطيع استخراج البيانات الأصلية

---

### 3. الحماية من الموقع المزيّف

#### صيغة Haversine
حساب المسافة الفعلية على سطح الكرة الأرضية — يصعب التلاعب فيه لأنه يُحسب على الخادم:

```php
function isWithinGeofence(float $empLat, float $empLon): array {
    // الخادم يحسب المسافة — لا يثق بالحسابات القادمة من الجهاز
    $distance = calculateDistance($empLat, $empLon, $workLat, $workLon);
    return ['allowed' => $distance <= $radius, 'distance' => $distance];
}
```

> الحسابات الجغرافية تتم على **الخادم**، ليس على المتصفح.

---

### 4. الحماية من الحقن (Injection)

#### SQL Injection
- جميع استعلامات DB تستخدم **Prepared Statements + PDO**
- `PDO::ATTR_EMULATE_PREPARES = false` ← Prepared Statements حقيقية

```php
// ✅ آمن دائماً
$stmt = db()->prepare("SELECT * FROM employees WHERE unique_token = ?");
$stmt->execute([$token]);
```

#### XSS (Cross-Site Scripting)
- دالة `sanitize()` على جميع المدخلات: `htmlspecialchars + strip_tags + trim`
- `htmlspecialchars()` على كل إخراج ديناميكي في HTML

---

### 5. حماية الـ API

- `Access-Control-Allow-Origin: [SITE_URL]` ← CORS محدود لنطاق الموقع فقط
- التحقق من `$_SERVER['REQUEST_METHOD'] === 'POST'`
- قراءة البيانات من `php://input` + `json_decode` (لا `$_POST` — أكثر أماناً مع JSON)
- التحقق المتسلسل: Token ← Geofence ← وقت الدوام ← تكرار

   - **المسمى الوظيفي** ← يظهر تحت الاسم
   - **PIN** ← رقم سري فريد (للرجوع إليه مستقبلاً)
   - **الفرع** ← اختر الفرع التابع له الموظف (يحدد نطاقه الجغرافي)
   - **رقم الواتساب** (اختياري) ← بصيغة دولية مثل `966501234567`
3. اضغط **"حفظ"** ← يُولّد الرابط الفريد تلقائياً

> **ملاحظة:** إذا اختير فرع للموظف، سيستخدم النظام إحداثيات الفرع ونصف قطره للتحقق من موقعه، بدلاً من الإعدادات العامة.

### إرسال الرابط للموظف

- **مع رقم واتساب:** اضغط زر **"💬 واتساب"** ← يفتح رسالة جاهزة
- **بدون رقم:** اضغط **"📋 نسخ"** ← انسخ الرابط وأرسله يدوياً

### إجراءات أخرى على الموظف

2. عندما تريد ربط جهازه، اضغط زر **🔑** (ربط الجهاز)

3. إذا أردت ربط الجهاز الجديد، اضغط **🔑** (ربط الجهاز)

### الموظف فقد رابطه

2. أرسل الرابط الجديد عبر واتساب

## 1. الملخص التنفيذي

### نظرة عامة
النظام مبني بـ PHP 8.3 Native + MySQL/MariaDB + Vanilla JavaScript. يخدم **40 موظفاً** في **5 فروع** بروابط فريدة (token-based). بعد جلسة إصلاح شاملة، ارتفع تقييم النظام من **7.2/10** إلى **9.0/10**.

### النقاط القوية ✅
- بنية كود منظمة (includes, api, admin, employee)
- 100% Prepared Statements ضد SQL Injection
- تشفير bcrypt للكلمات السرية
- حماية CSRF على جميع النماذج
- فهارس قاعدة بيانات محسّنة
- HTTPS + SameSite=Strict + CSP headers
- Brute Force protection (5 محاولات / 5 دقائق)
- Settings cache (تجنب N+1 queries)
- التحقق من أوقات الدوام في API

## 2. قائمة الإصلاحات المنفذة

### 🔴 Critical → ✅ تم الحل

### 🟡 Medium → ✅ تم الحل

| # | المشكلة | الحل المطبق | الملف |

| # | التحسين | الحل المطبق | الملف |
