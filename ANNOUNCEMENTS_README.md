# 📢 نظام الإعلانات والأخبار

## ✨ الميزات المضافة

### 1. شريط الأخبار المتحرك
- شريط أنيق يظهر تحت header بوابة الموظف
- تمرير تلقائي لا نهائي للإعلانات
- يتوقف عند التمرير بالماوس
- يدعم الأيقونات والألوان المخصصة
- يتكيف مع الشاشات المختلفة (Responsive)

### 2. صفحة إدارة الإعلانات (لوحة التحكم)
- واجهة أنيقة لإدارة الإعلانات والأخبار والتعميمات
- بطاقات إعلانات بتصميم عصري (Cards)
- إضافة/تعديل/حذف/تفعيل/تعطيل الإعلانات
- محدد أيقونات تفاعلي (Emoji Picker)
- محدد ألوان جميل
- فلترة وبحث متقدم
- تصنيف حسب النوع والأولوية
- جدولة الإعلانات (تاريخ بداية/نهاية)
- استهداف محدد (الجميع/فرع معين)
- إحصائيات فورية

### 3. API الإعلانات
- نقطة دخول: `/api/announcements.php`
- يدعم فلترة الإعلانات حسب:
  - الشريط المتحرك فقط (`ticker_only=1`)
  - الفرع (`branch_id`)
  - التاريخ (تلقائياً)
  - الجمهور المستهدف
- استجابة موحدة بتنسيق JSON

## 📁 الملفات المضافة/المعدلة

## 🗄️ هيكل قاعدة البيانات

جدول `announcements`:
- `id` - المعرف
- `title` - عنوان الإعلان
- `content` - محتوى الإعلان
- `type` - نوع (news, announcement, circular, alert, event)
- `priority` - أولوية (low, normal, high, urgent)
- `icon` - أيقونة Emoji
- `color` - لون HEX
- `is_active` - نشط/معطل
- `show_in_ticker` - عرض في الشريط المتحرك
- `start_date` - تاريخ البداية (اختياري)
- `end_date` - تاريخ النهاية (اختياري)
- `target_audience` - الجمهور المستهدف
- `branch_id` - الفرع المستهدف (اختياري)
- `created_by` - المستخدم الذي أنشأ الإعلان
- `created_at` - تاريخ الإنشاء
- `updated_at` - تاريخ آخر تحديث

## 🎯 الاستخدام

### إضافة إعلان جديد:
1. ادخل لوحة التحكم
2. اضغط على "الإعلانات والأخبار" من القائمة الجانبية
3. اضغط "+ إضافة إعلان"
4. املأ البيانات واختر الأيقونة واللون
5. حدد النوع والأولوية
6. اختر الجمهور المستهدف
7. فعّل "عرض في الشريط المتحرك" إذا أردت
8. احفظ

### عرض الإعلانات في الشريط:
- يتم جلب الإعلانات تلقائياً عند فتح بوابة الموظف
- تظهر فقط الإعلانات:
  - النشطة (is_active = 1)
  - المفعلة للشريط (show_in_ticker = 1)
  - ضمن نطاق التاريخ
  - المستهدفة للموظف (حسب الفرع)
- تحديث تلقائي كل 5 دقائق

## 📊 API Endpoints

### GET /api/announcements.php
جلب الإعلانات النشطة

**Parameters:**
- `ticker_only` (optional) - فقط إعلانات الشريط المتحرك (1/0)
- `branch_id` (optional) - تصفية حسب الفرع

**Response:**
```json
{
  "success": true,
  "data": {
    "announcements": [
      {
        "id": 1,
        "title": "عنوان الإعلان",
        "content": "محتوى الإعلان",
        "type": "announcement",
        "priority": "high",
        "icon": "📢",
        "color": "#F97316",
        "created_at": "2026-04-15 10:30:00",
        "priority_label": "مرتفع",
        "type_label": "إعلان"
      }
    ],
    "count": 1
  },
  "meta": {
    "timestamp": "2026-04-15T10:30:00+03:00"
  }
}
```

## 🔒 الأمان

- ✅ فحص CSRF Token في جميع العمليات
- ✅ فحص Origin/Referer في API
- ✅ تنظيف المدخلات (sanitize)
- ✅ Prepared Statements لحماية من SQL Injection
- ✅ XSS Protection عبر htmlspecialchars
- ✅ صلاحيات المدير فقط لإدارة الإعلانات

## 💡 نصائح

1. استخدم الأولوية "عاجل" للإعلانات المهمة فقط
2. اختر ألوان متناسقة مع هوية المنشأة
3. اجعل العنوان قصيراً ومعبراً (يظهر في الشريط)
4. استخدم الجدولة للإعلانات الموسمية
5. استهدف الفروع المحددة للإعلانات الخاصة
6. راجع الإعلانات المنتهية بانتظام

## 🐛 استكشاف الأخطاء

### الشريط المتحرك لا يظهر:
1. تأكد من وجود إعلانات نشطة مفعلة للشريط
2. تحقق من Console المتصفح للأخطاء
3. تأكد من صحة API endpoint

### الإعلانات لا تظهر للموظف:
1. تحقق من target_audience الإعلان
2. تحقق من branch_id إذا كان الاستهداف لفرع محدد
3. تأكد من التواريخ (start_date/end_date)

### مشاكل التنسيق:
1. امسح cache المتصفح
2. تحقق من تحميل radar.css
3. تحقق من النسخة في URL (?v=...)
