# نظام الحضور والانصراف الإلكتروني

**الإصدار:** 3.2.0 | **آخر تحديث:** مارس 2026

نظام ويب متكامل لتسجيل حضور وانصراف الموظفين عبر روابط فريدة (WhatsApp) مع التحقق من الموقع الجغرافي، بصمة الجهاز، والتحديث بالوقت الفعلي.

## المميزات الرئيسية

| الميزة | الوصف |
|--------|-------|
| تسجيل الحضور/الانصراف | عبر روابط فريدة لكل موظف (WhatsApp) |
| التحقق الجغرافي (Geofencing) | التأكد من تواجد الموظف ضمن نطاق العمل |
| بصمة الجهاز (Device Fingerprinting) | التحقق من الجهاز المسجل لكل موظف |

| التحديث بالوقت الفعلي | تحديث تلقائي للبيانات كل 15-20 ثانية عبر AJAX |
| مؤشر البث المباشر | مؤشر مرئي يوضح حالة الاتصال (مباشر / متوقف / خطأ) |
| تصدير CSV | تصدير تقارير الحضور بصيغة CSV |

## التقنيات المستخدمة

- **PHP 8.2+** Native (بدون إطارات عمل)
- **MySQL 8.0** / MariaDB
- **JavaScript** خالص (ES6+ بدون إطارات)
- **HTML5 + CSS3** مع تصميم RTL عربي
- **Geolocation API** للتحقق الجغرافي
- **Web Audio API** لصوت الرادار/السونار
- **Fetch API** للتحديث بالوقت الفعلي (AJAX Polling)
- **Leaflet.js** للخرائط التفاعلية (واجهة الموظف + الإدارة)
- **Service Worker + PWA** للتطبيق التقدمي
- **Apache / LiteSpeed** مع mod_rewrite
- **SVG** للأيقونات (بدون مكتبات خارجية)

## ما الجديد في الإصدار 3.0.0

### واجهة الموظف — تحسينات UI/UX
- **شعار مخصص (loogo.png)** يُعرض في كل مكان: الهيدر، الفافيكون، مركز الخريطة
- **خلفية مزخرفة بالشعار** — نمط متكرر متحرك (Watermark Pattern) بشفافية خفيفة
- **أنيميشن الشعار** — نبض (Pulse)، توهج (Glow)، تعويم (Float) في الهيدر والخريطة
- **صوت رادار/سونار** عند فتح الصفحة باستخدام Web Audio API (3 نبضات سونار)
- **خريطة تفاعلية Leaflet** تُظهر:
  - مركز العمل (بالشعار) مع دائرة نطاق السياج الجغرافي
  - دائرة عرض أكبر بـ 10 أمتار من نطاق تسجيل الحضور
  - موقع الموظف الحالي (نقطة زرقاء متحركة)
  - زر إظهار/إخفاء الخريطة
- **تنبيه GPS مستمر** — نافذة منبثقة مستمرة إذا لم يُفعّل الموقع:
  - زر لفتح إعدادات الموقع
  - زر لإعادة المحاولة
  - فحص دوري كل 3 ثوانٍ لحالة صلاحية GPS
  - اختفاء تلقائي عند تفعيل الموقع

### لوحة التحكم (الإدارة) — تحسينات الهاتف
- **تصميم متجاوب محسّن** — أزرار، جداول، بطاقات، نماذج مناسبة للهاتف
- **زر تسجيل خروج مرئي** في الشريط العلوي (Topbar)
- **إخفاء ذكي** — عناصر غير ضرورية تُخفى على الشاشات الصغيرة
- **فافيكون الشعار** — بدلاً من SVG مولّد
- **شعار الصفحة** — صورة حقيقية في الشريط الجانبي

### أمان (من v2.2.0)
- Rate Limiting (ملفي) لجميع APIs
- حماية Brute Force لتسجيل الدخول (جدول login_attempts)
- CSRF Token مع تدوير (Rotation)
- تعقيد كلمة المرور (8+ حروف، أرقام، رموز)
- ملف .env للـ Secrets (بدلاً من config.php مباشرة)
- حماية .htaccess متقدمة (CSP، X-Frame-Options)

### قاعدة البيانات (من v2.2.0)
- عمود `attendance_date` — فهرسة يومية سريعة
- عمود `late_minutes` — حساب التأخير تلقائياً
- عمود `deleted_at` — حذف ناعم للموظفين
- جدول `audit_log` — سجل تدقيق شامل
- جدول `login_attempts` — حماية محاولات الدخول
- جدول `leaves` — إدارة الإجازات

## خطوات التثبيت

### 1. المتطلبات
| المتطلب | الحد الأدنى | الموصى به |
|---------|-------------|-----------|
| PHP | 8.0 | 8.3+ |
| MySQL | 5.7 | 8.0+ |
| MariaDB | 10.4 | 10.11+ |
| Apache | 2.4 | 2.4+ |
| إضافات PHP | PDO, PDO_MySQL, session, json | + mbstring, openssl |

### 2. رفع الملفات
```bash
# انسخ المجلد إلى مجلد الويب
cp -r attendance-system/ /var/www/html/
# أو عبر FTP إلى public_html/
```

### 3. تعديل إعدادات الاتصال
افتح `includes/config.php` وعدّل:
```php
define('DB_HOST', 'localhost');
define('DB_USER', 'your_db_user');
define('DB_PASS', 'your_db_password');
define('DB_NAME', 'attendance_system');
define('SITE_URL', 'https://yourdomain.com/attendance-system');
```

كذلك في `install.php` (أول 4 سطور):
```php
define('DB_USER', 'your_db_user');
define('DB_PASS', 'your_db_password');
```

### 4. تشغيل سكريبت التثبيت
```
https://yourdomain.com/attendance-system/install.php
```
سيقوم بـ:
- إنشاء قاعدة البيانات والجداول (5 جداول: employees, attendances, admins, settings, branches)
- إدخال الموظفين تلقائياً مع روابط فريدة
- إنشاء حساب مدير افتراضي
- إنشاء ملف قفل `install.lock`

### 5. بعد التثبيت — احذف install.php فوراً!
```bash
rm public_html/attendance-system/install.php
```
أو تأكد من وجود `install.lock` لمنع إعادة التثبيت.

### 6. بيانات المدير الافتراضية
```
المستخدم: admin
كلمة المرور: Admin@1234
```
**غيّر كلمة المرور فوراً من الإعدادات > كلمة المرور!**

## إعداد Cron Jobs (مهمة المهام الآلية)

يحتاج النظام إلى cron job يعمل كل دقيقة لتنفيذ المهام الآلية التالية:

### 1. الانصراف التلقائي
يسجل انصراف تلقائي للموظفين الذين لم يسجلوا انصراف عند وصول وقت `check_out_end_time` المحدد في جدول الفرع.

### 2. إنهاء الدوام الإضافي التلقائي
ينهي sessions الدوام الإضافي المفتوحة تلقائياً عند الساعة 3:00 فجراً (موعد حفظ السجلات).

### إعداد Cron على السيرفر

#### cPanel / Hostinger
1. افتح لوحة تحكم cPanel → Cron Jobs
2. أضف cron job جديد:
   - الدقائق: `*` (كل دقيقة)
   - الساعات: `*`
   - أيام الشهر: `*`
   - الأشهر: `*`
   - أيام الأسبوع: `*`
   - الأمر:
     ```bash
     /usr/bin/php /path/to/attendance-system/cron/auto-checkout.php
     ```

#### LiteSpeed / Apache (crontab)
```bash
crontab -e
```
أضف السطر التالي:
```
* * * * * /usr/bin/php /home/username/public_html/attendance-system/cron/auto-checkout.php >> /home/username/cron.log 2>&1
```

#### للتشغيل اليدوي (اختبار)
```bash
php /path/to/attendance-system/cron/auto-checkout.php
```

أو عبر HTTP (لأغراض الاختبار فقط):
```
https://yourdomain.com/attendance-system/cron/auto-checkout.php?secret=auto_checkout_2026
```

## التحديث بالوقت الفعلي (Real-time Updates)

### آلية العمل
يستخدم النظام تقنية **AJAX Long Polling** عبر `fetch()` API لتحديث البيانات تلقائياً:

```
[المتصفح] ──fetch()──> [api/realtime-dashboard.php] ──SQL──> [MySQL]
     ↑                          │
     └───── JSON Response ──────┘
         (كل 15-20 ثانية)
```

### الصفحات المدعومة

| الصفحة | API Endpoint | فترة التحديث | البيانات المحدثة |
|--------|-------------|-------------|-----------------|
| لوحة التحكم | `/api/realtime-dashboard.php` | 15 ثانية | الإحصائيات + آخر التسجيلات + الغائبون |
| تقارير الحضور | `/api/realtime-attendance.php` | 20 ثانية | الإحصائيات + السجلات + الترقيم |

### مؤشر البث المباشر
يظهر مؤشر مرئي في الزاوية يوضح حالة الاتصال:
- **نقطة خضراء نابضة** + "مباشر": الاتصال نشط والبيانات محدثة
- **نقطة صفراء ثابتة** + "متوقف": الصفحة مخفية أو خطأ مؤقت
- **نقطة حمراء ثابتة** + "خطأ": فشل الاتصال 3 مرات متتالية

### تحسينات الأداء
- **توقف تلقائي**: يتوقف الاستطلاع عند إخفاء التبويب (`visibilitychange`)
- **استئناف فوري**: يُحدّث فوراً عند العودة للصفحة
- **عدد الأخطاء**: يتتبع عدد الأخطاء المتتالية قبل إظهار حالة "خطأ"
- **No-cache**: ترويسات عدم التخزين المؤقت لضمان بيانات طازجة
- **تحريك سلس**: تأثيرات CSS عند تغيّر القيم (flash animation)

### التخصيص
لتغيير فترة التحديث، عدّل المتغير في الصفحة:
```javascript
const REFRESH_INTERVAL = 15000; // بالميللي ثانية (15 ثانية)
```

## إعداد الموقع الجغرافي

### الإعدادات العامة
1. سجّل دخولك كمدير
2. اذهب إلى **الإعدادات** ← **الموقع الجغرافي**
3. اضغط **"موقعي الحالي"** أو انقر على الخريطة
4. حدد نصف قطر المنطقة المسموحة (مثلاً 200 متر)
5. احفظ الإعدادات

### إعدادات الفرع
كل فرع يمكن أن يكون له:
- إحداثيات مستقلة (خط العرض / خط الطول)
- نصف قطر جيوفينس مخصص
- مواعيد دوام مستقلة (بداية/نهاية)
- أوقات تسجيل حضور/انصراف مخصصة
- إعدادات دوام إضافي مستقلة

### ترتيب الأولوية
```
إحداثيات الفرع (إذا كان الموظف معين لفرع)
    ↓ fallback
الإعدادات العامة (work_latitude, work_longitude)
    ↓ fallback
مسموح بأي موقع (إذا كلا الإحداثيات = 0)
```

## بصمة الجهاز (Device Fingerprinting)

### آلية العمل (ربط يدوي فقط)
1. عند فتح الموظف لرابطه، يتم حساب بصمة الجهاز وإرسالها للخادم
2. **إذا لم يكن هناك ربط مفعّل:** الموظف يدخل بدون ربط جهاز
3. **إذا فعّل المشرف وضع الربط:** يُربط الجهاز عند الدخول التالي
4. بعد الربط، كل تسجيل لاحق يتحقق من بصمة الجهاز
5. إذا اختلفت البصمة، يُرفض الدخول

> **لا يحدث ربط تلقائي** — المشرف يتحكم بالكامل.

### إدارة الأجهزة
- **فك الربط:** يمكن للمشرف فك ربط الجهاز (يصبح الرابط حراً)
- **ربط جهاز جديد:** فك الربط أولاً ثم تفعيل وضع الربط
- **الربط اليدوي:** من صفحة الموظفين → زر 🔑 (ينتظر الدخول التالي)

## API Reference

### POST `/api/check-in.php` — تسجيل الحضور
**الطلب:**

## دليل صفحات لوحة التحكم

### لوحة التحكم (`dashboard.php`)
- **5 بطاقات إحصائية**: الفروع النشطة، إجمالي الموظفين، الحاضرون، المنصرفون، الغائبون
- **آخر 15 تسجيل**: جدول يعرض أحدث الدخول/الانصراف
- **الغائبون اليوم**: قائمة أول 10 موظفين غائبين
- **تحديث تلقائي** كل 15 ثانية مع مؤشر "مباشر"
- **تأثيرات بصرية**: وميض عند تغيّر القيم، انزلاق للسجلات الجديدة

### إدارة الفروع (`branches.php`)
- **إضافة / تعديل / حذف** فروع
- **تفعيل / تعطيل** فروع
- كل فرع يتضمن:
  - اسم + إحداثيات + نصف قطر جيوفينس
  - مواعيد الدوام (بداية/نهاية)
  - أوقات تسجيل الحضور والانصراف
  - إعدادات الدوام الإضافي
  - عدد الموظفين المعينين

### إدارة الموظفين (`employees.php`)
- **إضافة / تعديل / حذف** موظفين
- **تفعيل / تعطيل** حسابات
- **إعادة توليد** رابط الحضور (token جديد)
- **إعادة تعيين** بصمة الجهاز
- **نسخ** رابط الحضور
- **إرسال** عبر واتساب
- تعيين موظف لفرع

### تقارير الحضور (`attendance.php`)
- **فلاتر متقدمة**: تاريخ من/إلى، موظف، فرع، نوع (دخول/انصراف)
- **4 بطاقات إحصائية** للفترة المحددة
- **تصدير CSV** بصيغة Excel-compatible (مع BOM UTF-8)
- **أزرار خريطة** لعرض موقع كل تسجيل
- **ترقيم الصفحات** (25 سجل/صفحة)
- **تحديث تلقائي** كل 20 ثانية مع مؤشر "مباشر"

### تقرير التأخير (`late-report.php`) 🆕
- **فلاتر متقدمة**: موظف محدد / فرع / جميع الموظفين + فترة زمنية
- **حساب دقيق للتأخير**: على مستوى الدقيقة بمقارنة وقت check-in مع check_in_end_time
- **4 بطاقات إحصائية**: مجموع التأخير، عدد أيام التأخير، الموظفون المتأخرون، متوسط التأخير
- **ملخص حسب الموظف**: عدد أيام التأخير + مجموع الدقائق + متوسط التأخير لكل موظف
- **تقرير تفصيلي يومي**: التاريخ، الموظف، الفرع، وقت الحضور، الوقت المتوقع، مدة التأخير
- **تصنيف ملون**: تأخير منخفض (<15د) / متوسط (15-45د) / عالي (>45د)

### الإعدادات (`settings.php`)
5 تبويبات:
1. **الموقع الجغرافي**: خريطة Leaflet تفاعلية + نصف القطر
2. **أوقات الدوام**: بداية/نهاية الدوام + مواعيد التسجيل + إظهار الانصراف
3. **الدوام الإضافي**: تفعيل/تعطيل + حد أدنى + بدء بعد
4. **إعدادات عامة**: اسم النظام + الشركة + المنطقة الزمنية
5. **كلمة المرور**: تغيير كلمة مرور المدير

## الأمان

| الميزة | التفاصيل |
|--------|----------|
| SQL Injection | Prepared Statements (PDO) في جميع الاستعلامات |
| XSS | `htmlspecialchars()` + `strip_tags()` على جميع المدخلات |
| CSRF | Token عشوائي 64 حرف مع `hash_equals()` |
| كلمات المرور | bcrypt عبر `password_hash()` / `password_verify()` |
| تجديد الجلسة | `session_regenerate_id(true)` عند تسجيل الدخول |
| Token الموظف | 64 حرف hex عشوائي (CSPRNG) لكل موظف |
| حماية includes | `.htaccess` → HTTP 403 |
| Cookie Security | HttpOnly + SameSite=Strict |
| Brute Force | 5 محاولات خلال 5 دقائق ثم قفل 5 دقائق |
| CSP Header | Content-Security-Policy |
| X-Frame-Options | SAMEORIGIN |
| Permissions-Policy | geolocation=(self) |
| حماية الملفات | .env, .sql, .zip, .log محمية عبر .htaccess |
| فحص وقت الدوام | التحقق من المواعيد في API check-in/out |
| فحص is_active | التحقق عند كل طلب API |
| Real-time Auth | فحص جلسة المدير في كل طلب API مباشر |
| No-Cache Headers | ترويسات عدم التخزين على API endpoints المباشرة |

## استكشاف الأخطاء

| المشكلة | الحل |
|---------|------|
| الموقع لا يُحدَّد | تأكد من HTTPS (أو localhost)، وأن المتصفح يسمح بالوصول للموقع |
| خطأ في قاعدة البيانات | تحقق من `includes/config.php` وصلاحيات المستخدم |
| صفحات 404 | تأكد من تفعيل mod_rewrite وأن `.htaccess` مفعّل |
| بصمة الجهاز مرفوضة | أعد تعيين بصمة الجهاز من إدارة الموظفين |
| مؤشر "مباشر" يظهر "خطأ" | تحقق من الاتصال بالخادم واتصال قاعدة البيانات |
| لا يعمل على الموبايل | تأكد من استخدام HTTPS (مطلوب لـ Geolocation API) |
| الانصراف لا يظهر | تحقق من إعدادات `checkout_show_before` و `check_out_start_time` |
| الدوام الإضافي لا يعمل | تأكد من `allow_overtime = 1` وأن شروط الوقت محققة |
| التحديث التلقائي بطيء | عدّل `REFRESH_INTERVAL` في JavaScript (الافتراضي 15-20 ثانية) |
