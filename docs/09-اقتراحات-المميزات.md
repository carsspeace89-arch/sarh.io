## 🔴 الأولوية العاجلة (يجب تنفيذها قريباً)

### 1. تطبيق HTTPS إلزامي
**المشكلة:** النظام لا يُجبر على HTTPS، مما يُعرّض بيانات المصادقة للاعتراض.  
**الحل المقترح:**
```php
// في includes/config.php
if (!isset($_SERVER['HTTPS']) || $_SERVER['HTTPS'] !== 'on') {
    header('Location: https://' . $_SERVER['HTTP_HOST'] . $_SERVER['REQUEST_URI'], true, 301);
    exit;
}
```
**الأثر:** حماية التوكنات والجلسات من Man-in-the-Middle.

---

### 2. تحديد معدل الطلبات لكل موظف (Per-Employee Rate Limiting)
**المشكلة:** الحد الحالي مبني على IP فقط — موظفون خلف NAT يشتركون في نفس IP.  
**الحل المقترح:** إضافة rate limiting بالـ `unique_token` في `api/check-in.php`:
```php
$rl = new \App\Middleware\RateLimiter();
$result = $rl->checkByToken($token, 'check-in', 10, 60); // 10 طلبات/دقيقة
if (!$result['allowed']) {
    jsonResponse(['success' => false, 'message' => 'حاول مرة أخرى بعد قليل'], 429);
}
```

---

### 3. التحقق من دقة GPS
**المشكلة:** الموظف يمكنه إرسال إحداثيات بدقة 0.001م وهو في الواقع بعيد.  
**الحل المقترح:** رفض التسجيل إذا كانت دقة GPS أكثر من 100 متر:
```php
if (isset($data['accuracy']) && (float)$data['accuracy'] > 100) {
    jsonResponse(['success' => false, 'message' => 'دقة GPS ضعيفة، حاول في مكان مكشوف'], 400);
}
```

## 🟡 الأولوية المتوسطة (خلال الأشهر القادمة)

### 4. إشعارات واتساب التلقائية للموظفين
**الفكرة:** عند تسجيل الحضور أو الانصراف، يصل إشعار واتساب للموظف:
- "تم تسجيل حضورك الساعة 08:02 ✅"
- "تم تسجيل انصرافك الساعة 16:58 ✅"
- "تنبيه: لم يُسجَّل حضورك اليوم ⚠️" (إشعار التأخر)

**الأدوات المتوفرة فعلاً:** `api/whatsapp.php` موجود — يحتاج فقط تشغيل تلقائي عبر cron.

---

### 5. لوحة تحليلات متقدمة (Analytics Dashboard)
**الفكرة:** إضافة رسوم بيانية تفاعلية لـ:
- معدل الحضور الأسبوعي/الشهري لكل فرع
- موظفو التأخر المتكرر (أكثر من 3 مرات/شهر)
- توزيع ساعات العمل الإضافية
- خريطة حرارية للحضور (heatmap)

**مكتبة مقترحة:** Chart.js أو ApexCharts (تشتغل بدون backend إضافي).

---

### 6. نظام إجازات متكامل
**الوضع الحالي:** إضافة إجازات يدوية فقط من المشرف.  
**المطلوب:**
- طلب إجازة من الموظف عبر الهاتف
- موافقة/رفض من المشرف
- رصيد إجازات (سنوية/مرضية/طارئة)
- تقرير رصيد الإجازات المتبقية

---

### 7. تصدير تقارير متقدمة
**الوضع الحالي:** تصدير CSV بسيط.  
**المطلوب:**
- تصدير PDF مُنسَّق بشعار الشركة
- تقرير الرواتب (نموذج مبني على ساعات الحضور + الأجر الساعي)
- تقرير العمل الإضافي المعتمد شهرياً
- إرسال التقارير تلقائياً بالبريد الإلكتروني

---

### 8. نظام PIN محسّن (6 أرقام + تجميل)
**المشكلة الحالية:** PIN مكون من 4 أرقام فقط — ضعيف أمنياً (10,000 احتمال).  
**الحل المقترح:**
- رفع PIN إلى 6 أرقام (فقط للمواقع الجديدة)
- إضافة قفل بعد 3 محاولات خاطئة لـ 15 دقيقة
- خيار بصمة الإصبع / Face ID في الأجهزة الداعمة (Web Authentication API)

## 🟢 الأولوية المنخفضة (تطويرات مستقبلية)

### 9. تطبيق موبايل (Progressive Web App محسّن)
**الوضع الحالي:** PWA بسيط — يعمل في المتصفح.  
**التحسينات المقترحة:**
- Push Notifications بدون فتح التطبيق
- تسجيل الحضور بضغطة واحدة (one-tap check-in)
- وضع offline مع مزامنة عند الاتصال
- Widget للشاشة الرئيسية (iOS/Android)

---

### 10. تعدد المشرفين مع صلاحيات دقيقة
**المشكلة الحالية:** مشرف واحد فقط يملك كل الصلاحيات.  
**الحل المقترح:**
```
أدوار مقترحة:
  - super_admin: كل الصلاحيات
  - branch_manager: صلاحيات فرعه فقط
  - hr_viewer: يشاهد التقارير فقط بدون تعديل
  - auditor: يشاهد سجل التدقيق فقط
```

---

### 11. التكامل مع أنظمة الرواتب
**الفكرة:** تصدير بيانات الحضور بصيغ مدعومة من:
- مسير (Maseer) — نظام رواتب سعودي
- SAP HR
- Oracle HRMS
- صيغة Excel/CSV مخصصة لكل نظام

### 12. نظام إشعارات بريد إلكتروني
**الفكرة:**
- بريد يومي للمشرف بملخص حضور الفروع
- تنبيه فوري عند تسجيل مشبوه (tampering_cases)
- تقرير أسبوعي PDF مرفق في بريد

---

### 13. API عامة للتكامل الخارجي
**الفكرة:** توفير REST API موثّقة بـ OAuth2 لربط النظام مع:
- نظام بوابة الشركة الداخلية
- جداول الوردية من نظام خارجي
- نظام ERP/CRM

---

### 14. نسخ احتياطي تلقائي مجدوَل
**الوضع الحالي:** لا يوجد نسخ احتياطي تلقائي.  
**الحل المقترح:**
```bash
# cron يعمل يومياً عند 2:00 صباحاً
0 2 * * * /usr/bin/php /path/to/cron/backup.php
```
- حفظ النسخة في مجلد `storage/backups/` مع ضغط gzip
- الاحتفاظ بآخر 30 نسخة
- إرسال رسالة واتساب بنجاح/فشل النسخ

---

### 15. واجهة إدارة متعددة اللغات
**الوضع الحالي:** عربي فقط.  
**الحل المقترح:** نظام i18n بسيط:
```php
$lang = include "lang/{$_SESSION['lang']}.php"; // ar.php, en.php
echo $lang['attendance']['title'];
```
