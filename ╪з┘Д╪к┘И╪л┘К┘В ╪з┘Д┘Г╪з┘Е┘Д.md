## 1 — ملخص المشروع

نظام ويب متكامل لتسجيل **حضور وانصراف الموظفين** عبر روابط WhatsApp فريدة، مع التحقق من الموقع الجغرافي المعزز بتقييم المخاطر (Risk-Based Geofencing) وبصمة الجهاز، يدعم فروعاً متعددة وورديات متعددة لكل فرع، مع واجهة إدارية كاملة ولوحة متابعة لحظية، ونظام صفوف غير متزامن (Queue System)، وتسجيل منظم (Structured Logging)، وواجهة API مُنسّخة (Versioned API v1).

**التقنيات:**
| المكوّن | التفصيل |
|---------|--------|
| PHP | 8.2+ Native — Clean MVC + Service-Oriented Architecture |
| قاعدة البيانات | MySQL 8.0 / MariaDB — utf8mb4_unicode_ci |
| التخزين المؤقت | Redis (اختياري) + ملفات (fallback تلقائي) |
| الصفوف | Queue System — Redis driver أو Database driver |
| السجلات | Structured JSON Logging — PSR-3 متوافق |
| JavaScript | Vanilla ES6+، Leaflet.js، Web Audio API، SubtleCrypto API |
| CSS | Custom RTL عربي، Dark Mode، متجاوب |
| خادم الويب | Apache / LiteSpeed + mod_rewrite |
| PWA | manifest.json + Service Worker |
| التوقيت | Asia/Riyadh (UTC+3) |
| Multi-Tenant | مُجهّز (tenant_id injection تلقائي في Model) |

4. يُحقن في `$_ENV` و `putenv()` (فقط إذا لم يكن معرّفاً)
5. `Config::loadDefaults()`: يبني المصفوفة الهرمية مع استخدام `$_ENV` كأولوية

### سياق الطلب: `src/Core/RequestContext.php`

يُنشأ تلقائياً عند كل طلب عبر `RequestContext::init()` في `bootstrap.php`:

```php
RequestContext::init();           // يُنشئ request_id + يُحمّل tenant
RequestContext::id();             // → '3a7f1b2c9d4e5f06' (16 hex)
RequestContext::tenantId();       // → null أو int
RequestContext::setTenantId(5);   // تعيين المستأجر يدوياً
RequestContext::elapsed();        // → 0.0342 (ثوانٍ)
RequestContext::ip();             // → '192.168.1.1' (مع حماية proxy)
RequestContext::userAgent();      // → 'Mozilla/5.0...'
RequestContext::method();         // → 'POST'
RequestContext::uri();            // → '/api/v1/attendance/record'
RequestContext::toArray();        // مصفوفة كاملة
```

**حماية IP:**  
`X-Forwarded-For` و `HTTP_CLIENT_IP` يُقبلان فقط من بروكسيات موثوقة: `['127.0.0.1', '::1']`.  
يتم توثيق `FILTER_FLAG_NO_PRIVATE_RANGE | FILTER_FLAG_NO_RES_RANGE`.

### القسم A: الدوال المساعدة الدائمة (Pure Helpers)

هذه الدوال تبقى كما هي — مساعدة صغيرة لا تعتمد على أي خدمة:

```php
sanitize(string $value): string
```
`htmlspecialchars(strip_tags(trim($value)), ENT_QUOTES, 'UTF-8')` — تنظيف المدخلات.

```php
jsonResponse(array $data, int $code = 200): void
```
يُرسل رداً JSON مع `Content-Type: application/json` ويُنهي التنفيذ.

```php
arabicDateTime(string $datetime): string
```
يحوّل `datetime` إلى صيغة عربية: `YYYY/MM/DD - HH:MM AM/PM`.

```php
getClientIP(): string
```


### `GeofenceServiceV2` (جديد v5.0 — تقييم مخاطر شامل)

يحل محل `GeofenceService` القديم. يجمع 5 فحوصات مستقلة بنظام تقييم مخاطر تراكمي:

```php
class GeofenceServiceV2 {
    static calculateDistance(float $lat1, float $lon1, float $lat2, float $lon2): float
    // Haversine: R = 6,371,000 م

    validate(float $empLat, float $empLon, int $employeeId, ?int $branchId = null,
             ?string $fingerprint = null, float $accuracy = 0): array
    // → ['allowed', 'distance', 'radius', 'risk_score', 'risk_factors', 'message']

    isWithinGeofence(float $empLat, float $empLon, ?int $branchId = null): array
    // backward-compat: GPS only, بدون risk scoring
}
```

**الفحوصات الخمسة في `validate()`:**

| # | الفحص | النقاط | التفاصيل |
|---|-------|--------|---------|
| 1 | **GPS Geofence** | 0-40 | خارج النطاق = +40 نقطة |
| 2 | **GPS Accuracy** | 0-15 | دقة >500م = +15، >200م = +5 |
| 3 | **Device Consistency** | 0-20 | جهاز جديد = +20، أول تسجيل = +5 |
| 4 | **IP Consistency** | 0-10 | IP خاص = +5، proxy = +10 |
| 5 | **Velocity Check** | 0-25 | سفر مستحيل (>150 كم/ساعة) = +25 |

**العتبة:** `Config::get('geofence.risk_threshold', 70)` — فوق هذه القيمة يُرفض التسجيل.

**عند risk_score ≥ 50:** يُسجّل في `tampering_log` (حتى لو مسموح).

**Device Consistency:**
- يُخزّن أجهزة الموظف كـ JSON Array (حتى 5 أجهزة)
- أول جهاز: +5 (تسجيل أولي)
- جهاز معروف: 0 نقاط
- جهاز جديد: +20 نقطة + تسجيل أمني

**Velocity Check:**
- يجلب آخر موقع من `attendances`
- يحسب المسافة والزمن المنقضي
- إذا السرعة المطلوبة > 150 كم/ساعة والمسافة > 5 كم → مستحيل

## 11 — نظام الصفوف — src/Queue/

### المعماريّة

```
QueueManager (مدير مركزي)
├── Redis Driver: rPush/lPop + Sorted Set للتأخير
└── Database Driver: جدول jobs + FOR UPDATE SKIP LOCKED
         ↓
Job (كلاس أساسي مجرد)
├── SendWhatsAppJob
├── AutoCheckoutJob
└── ExportReportJob
         ↓
queue-worker.php (CLI Worker)
```

### `Job` — الكلاس الأساسي المجرد
```php
abstract class Job {
    public int $maxAttempts = 3;
    public int $retryAfter = 60;

    abstract handle(): void;           // منطق المهمة
    failed(\Throwable $e): void;       // hook عند الفشل النهائي
    serialize(): string;               // PHP serialize
    static deserialize(string $data): static;
}
```

### `QueueManager` — مدير الصفوف
```php
class QueueManager {
    static getInstance(): self           // Singleton

    push(Job $job, string $queue = 'default', int $delay = 0): int|string
    // يضع المهمة في الصف — delay بالثواني

    pop(string $queue = 'default'): ?array
    // يجلب المهمة التالية المستحقة
    // → ['id', 'class', 'data', 'attempts']

    complete(int|string $jobId): void    // يُعلّمها مكتملة
    fail(int|string $jobId, string $error, int $attempts): void
    // < maxAttempts: يُعيد الجدولة بعد retry_after
    // ≥ maxAttempts: يُعلّمها فاشلة نهائياً

    pendingCount(string $queue = 'default'): int
}
```

**Redis Driver:**
- `push()`: `rPush("queue:{name}", JSON)` — أو `zAdd` إذا delay > 0
- `pop()`: ينقل delayed المستحقة أولاً → `lPop("queue:{name}")`
- Payload: `{id, class, data(base64), attempts, created_at}`
- المهام المكتملة تُحذف تلقائياً

**Database Driver:**
- `push()`: `INSERT INTO jobs` مع `available_at`
- `pop()`: `SELECT ... FOR UPDATE SKIP LOCKED` → `UPDATE status='processing'` (atomic)
- `fail()`: إما يُعيد الجدولة (`status='pending'`, `available_at += retry_after`) أو `status='failed'`

## 16 — نظام الورديات

```
branches
  └── branch_shifts (1:N, حد 3 ورديات)
        ├── shift_number: 1 | 2 | 3
        ├── shift_start: TIME
        ├── shift_end: TIME
        └── is_active: 0 | 1
```

**آلية الكشف (`ShiftService::detectActiveShift()`):**
1. احسب `nowMin = الساعة × 60 + الدقيقة`
2. لكل وردية: `early = (shift_start - 60 + 1440) % 1440`
3. إذا `shift_end < early` → تعبر منتصف الليل: فحص `nowMin >= early || nowMin <= end`
4. وإلا: فحص `nowMin >= early && nowMin <= end`
5. إذا لا توجد وردية نشطة → أقرب وردية قادمة (أقل مسافة زمنية)
6. Fallback نهائي: الوردية الأولى أو `null`

**حساب التأخير (`ShiftService::calculateLateMinutes()`):**
- يجلب `work_start_time` من الوردية النشطة
- يحسب الفرق بالدقائق بين الآن و `work_start_time`
- يدعم عبور منتصف الليل (إذا `workStart > now + 12h` يرجع يوماً)

## 31 — نظام التحقق — Validator

### `src/Core/Validator.php`

واجهة Fluent API للتحقق من المدخلات تُستخدم من كل المتحكمات عبر `$this->validate()`.

```php
$validator = new Validator($data);
$validator->field('email')->required()->email();
$validator->field('name')->required()->string(2, 100);
$validator->field('age')->integer(18, 100);
$validator->field('pin')->pin();              // رقمي 4 خانات
$validator->field('lat')->latitude();         // -90 إلى 90
$validator->field('lon')->longitude();        // -180 إلى 180
$validator->field('token')->token();          // hex 64 حرف
$validator->field('type')->in(['in', 'out']); // قيمة من قائمة
$validator->field('date')->date();            // YYYY-MM-DD
$validator->field('time')->time();            // HH:MM أو HH:MM:SS

if ($validator->fails()) {
    // $validator->errors()     → ['field' => 'رسالة الخطأ', ...]
    // $validator->firstError() → 'رسالة أول خطأ'
}

$clean = $validator->validated();    // الحقول التي تم التحقق منها فقط
$all = $validator->allValidated();   // كل الحقول شاملة الاختيارية
```
